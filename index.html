Fuse Brand Bible — SharePoint Setup Guide

Step-by-step instructions for your marketing team to publish and maintain the brand bible on SharePoint

What this guide covers: How to build the Fuse Brand Bible as a SharePoint modern communication site — no code required. Your marketing team can edit every section directly in SharePoint's page editor.

Estimated setup time: 2–3 hours for initial build · Ongoing edits: 5–10 minutes per update

Phase 1

Create the SharePoint Communication Site

1

Create a new Communication Site

  • Go to your Microsoft 365 home → click SharePoint
  • Click "+ Create site" → select "Communication site"
  • Choose the "Blank" layout (gives you the most control)
  • Site name: Fuse Brand Bible
  • Site address: fuse-brand-bible (this becomes your URL)
  • Language: English (Australia)
  • Click Finish
💡 Your URL will be: https://[yourtenant].sharepoint.com/sites/fuse-brand-bible — share this with anyone who needs access.
2

Apply Fuse brand colours to the site

  • Click the Settings gear (top right) → "Change the look"
  • Click "Theme" → select "Custom" or the closest dark theme
  • Set the primary colour to Fuse Teal: #00ABAB
  • Set the accent colour to Fuse Magenta: #A72886
  • Under "Header" → choose "Compact" layout
  • Upload the Fuse logo (teal + Technology version, PNG) as the site logo
💡 Use the white reversed logo (white wordmark) if your header background is dark navy — it will show clearly against the dark background.
3

Set up site navigation

Edit the left or top navigation to mirror the brand bible sections. Go to Settings → Navigation and add these links:

  • Brand Essence
  • Positioning
  • Messaging Pillars
  • Logo System
  • Colour Palette
  • Typography
  • Imagery
  • Voice & Tone
  • Language Rules
  • Campaign Framework
  • Social Media
  • Digital Applications
  • Collateral
  • Do's & Don'ts
💡 Each navigation item should link to a separate SharePoint page within the same site — one page per section makes editing much easier.
Phase 2

Build Each Page Using SharePoint Web Parts

Every section of the brand bible maps to SharePoint web parts your team can drag, drop, and edit without any code. Here's the mapping:

Text web part
Body copy, rules, guidelines, definitions. Most content lives here.
Image web part
Logo variants, photography examples, colour swatches as images.
Hero web part
Section cover — navy/teal background with white heading. Use for page tops.
Columns web part
Side-by-side layouts — Do's vs Don'ts, pillar sub-points, tone examples.
Quick Links web part
Logo asset downloads, template downloads, resource links.
Highlighted content
Campaign cards — automatically surfaces latest campaign documents.
Divider web part
Visual section breaks between content blocks.
File viewer
Embed the brand bible PDF or presentation directly on the page.

Phase 3

Upload Logo Assets to SharePoint

4

Create a Logo Assets document library

  • In your brand bible site → click "+ New" → "Document library"
  • Name it: Brand Assets
  • Create folders: Logos / Templates / Photography / Icons
  • Upload all 8 logo SVG files into the Logos folder with clear names:
  • fuse-logo-teal-technology.svg — A1
  • fuse-logo-magenta-technology.svg — A2
  • fuse-logo-black-technology.svg — A3
  • fuse-logo-gray-technology.svg — A4
  • fuse-logo-teal.svg — B1
  • fuse-logo-magenta.svg — B2
  • fuse-logo-black.svg — B3
  • fuse-logo-gray.svg — B4
  • fuse-logo-white-technology.svg — AR reversed
  • fuse-logo-white.svg — BR reversed
💡 Use the Quick Links web part on the Logo System page to link directly to each file in this library — team members can download the exact file they need with one click.
Phase 4

Set Permissions

Brand bible permissions should be open for viewing but controlled for editing — you don't want anyone accidentally changing guidelines.

Site Owner
Marketing Lead / Brand Manager
Full control — can edit all pages, manage permissions, publish changes.
Member
Marketing Team
Can edit page content and upload assets. Cannot change site structure or permissions.
Visitor
All Fuse staff + external agencies
Read-only. Can view all pages and download assets. Cannot edit anything.
5

Configure permissions

  • Go to Site Settings → Site Permissions
  • Add the marketing team as Members
  • Share the site link with all Fuse staff — they'll auto-join as Visitors
  • For external agencies: share specific pages only using "Share with specific people" on individual pages
⚠️ Do not set the site to "Anyone with the link" — this makes it public to the internet. Keep it within your Microsoft 365 tenant, or share specific pages with named external users only.
Phase 5

Ongoing Maintenance Checklist

Give this checklist to whoever owns the brand bible going forward.

When to update the brand bible

  • New campaign launches → update the Campaign Framework page with new brief
  • New logo variant created → upload to Brand Assets library + update Logo System page
  • Social media strategy changes → update Social Media page content mix and specs
  • New product or service → add to Messaging Pillars if it shifts positioning
  • Rebrand or colour update → update Colour Palette page + re-export logo swatches
  • New team member onboarded → share the brand bible site link on day one
  • External agency briefed → share Visitor access to the full site or specific pages

How to edit a page (quick reference for marketing team)

  • Navigate to the page you want to edit
  • Click "Edit" in the top right of the page
  • Click on any text block to edit copy directly — it works like Word
  • To add a new section: click the "+" between sections → choose a web part
  • When finished: click "Republish" to make changes live immediately
  • To revert a change: click "Page details" → "Version history" → restore previous version
💡 SharePoint saves version history automatically — you can always roll back any accidental change. No changes are permanent until you click Republish.
